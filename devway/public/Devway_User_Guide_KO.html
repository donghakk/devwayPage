1. Blog

1-1. 사전 준비 사항

  • 네이버 개발자 센터에서 애플리케이션을 등록하고 클라이언트 아이디와 클라이언트 시크릿을 발급 받아야 합니다.

1-2. 설명

  • BlogProperties
    요청에 필요한 클라이언트 아이디, 파라미터 등을 설정하는 클래스입니다.
    • clientId
      • 발급받은 클라이언트 아이디를 입력합니다.
      • 보안을 위해 dotenv 등의 환경변수 관리를 권장합니다.
    • clientSecretkey
      • 클라이언트 시크릿을 입력합니다.
      • 보안을 위해 dotenv 등의 환경변수 관리를 권장합니다.
    • url
      • 요청을 보내는 URL 입니다.
    • query
      • 검색어
      • UTF-8로 인코딩 되어야 합니다.
    • display
      • 한 번에 표시할 검색 결과 개수
      • 기본 값은 10, 최댓값은 100 입니다.
    • start
      • 검색 시작 위치
      • 기본 값은 1, 최댓값은 1000 입니다.
    • sort
      • 검색 결과 정렬 방법
      • 기본 값은 SORT_BY_ACCURACY 입니다.
        • SORT_BY_ACCURACY으로 입력 시 정확도 순으로 내림차순 정렬 됩니다.
        • SORT_BY_LATEST으로 입력 시 날짜 순으로 내림차순 정렬 됩니다.
  • BlogResponseDTO
    네이버 블로그 검색 결과 중 원하는 응답 값만 선택하여 받아올 수 있는 클래스입니다.
    • title
      • 블로그 포스트의 제목
      • 제목에서 검색어와 일치하는 부분은 <b> 태그로 감싸져 있습니다.
    • link
      • 블로그 포스트의 URL
    • description
      • 블로그 포스트의 내용을 요약한 패시지 정보
      • 패시지 정보에서 검색어와 일치하는 부분은 <b> 태그로 감싸져 있습니다.
    • dateTime
      • 블로그 포스트가 작성된 날짜
  • BlogBlock
    API 요청을 보내는 비즈니스 로직이 포함된 클래스입니다.
    • 현재는 JSON 형식으로 응답값이 반환됩니다. 만약 XML 형식으로 응답값을 반환받고 싶다면, createHeaders() 메서드의 headers.setContentType(MediaType.APPLICATION_JSON)를

      headers.setContentType(MediaType.APPLICATION_XML)로 수정합니다.

      • .wav 음성 파일의 경로를 입력 받아 분석된 텍스트 저장
      • fileName 매개 변수에 음성 파일의 경로를 입력
      • 멤버 변수 speechTexts에 음성에서 분석된 텍스트 저장
    • speechTexts

    • KR__ALPHABET_FEMALE

Text API

1. Text API

1-1. 설명

  • TextBlock
    클래스 자료형으로 문자열을 입력 받아 다양한 텍스트 검사를 수행합니다.
    • 생성자
      • content 매개 변수에 String 자료형의 텍스트를 입력 받아 인스턴스를 생성합니다.
    • setContent
      • 문자열의 길이를 제한하여 content 값을 설정합니다.
    • setLengthLimit
      • 허용되는 문자열의 최대 길이를 설정합니다.
    • getContentLength
      • 현재 content의 문자열 길이를 반환합니다.
    • isLengthOver
      • 주어진 길이보다 content의 길이가 긴지 검사합니다.
    • onlyKorean
      • content가 한글만으로 구성되어 있는지 검사합니다.
    • isOnlyLowercaseEnglish
      • content가 소문자 영어만으로 구성되어 있는지 검사합니다.
    • isOnlyUppercaseEnglish
      • content가 대문자 영어만으로 구성되어 있는지 검사합니다.
    • isOnlyEnglish
      • content가 영어만으로 구성되어 있는지 검사합니다.
    • isBlank
      • content가 비어 있는지 검사합니다.
    • containsSpecialCharacters
      • content가 특수 문자를 포함하고 있는지 검사합니다.
    • doesNotContainSpecialCharacters
      • content가 특수 문자를 포함하지 않는지 검사합니다.
    • containsNoCharacters
      • content가 비어 있는지 검사합니다.
    • matchesPattern
      • 주어진 정규 표현식과 content가 일치하는지 검사합니다.

1-2. 사용 예시

  • Text API 연결 및 텍스트 처리
    • TextBlock 인스턴스 생성
    java코드 복사
TextBlock textBlock = new TextBlock("Hello, DevWay!");
  • 문자열 설정
    • setContent 메서드로 content 값 설정
    java코드 복사
textBlock.setContent("Welcome to DevWay");
  • 문자열 길이 검사
    • getContentLength 메서드로 문자열 길이 가져오기
    java코드 복사
int length = textBlock.getContentLength();
System.out.println("Content Length: " + length);
  • 문자열 패턴 검사
    • onlyKorean 메서드로 문자열이 한글인지 검사
    java코드 복사
boolean isKorean = textBlock.onlyKorean();
System.out.println("Is Korean: " + isKorean);
  • 특수 문자 포함 여부 검사
    • containsSpecialCharacters 메서드로 특수 문자가 포함되어 있는지 검사
    java코드 복사

System.out.println("Contains Special Characters: " + hasSpecialChars);
