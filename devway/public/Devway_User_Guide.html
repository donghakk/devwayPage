1. Blog

1-1. Prerequisites

  • You need to register your application and get a client ID and client secret from the Naver Developer Center.

1-2. Description

  • BlogProperties
    This class sets the client ID and parameters required for the request.
    • clientId
      • Enter the issued client ID.
      • It is recommended to manage environment variables such as dotenv for security.
    • clientSecretkey
      • Enter the client secret.
      • It is recommended to manage environment variables such as dotenv for security.
    • url
      • The URL to send the request.
    • query
      • Search term
      • Must be UTF-8 encoded.
    • display
      • The number of search results displayed at once
      • Default value is 10, maximum value is 100.
    • start
      • Search start position
      • Default value is 1, maximum value is 1000.
    • sort
      • Method of sorting search results
      • The default is SORT_BY_ACCURACY.
        • If entered as SORT_BY_ACCURACY, it is sorted in descending order by accuracy.
        • If entered as SORT_BY_LATEST, it is sorted in descending order by date.
  • BlogResponseDTO
    This class can select and receive only the desired response values from the Naver Blog search results.
    • title
      • Title of the blog post
      • The part that matches the search term in the title is enclosed in <b> tags.
    • link
      • URL of the blog post
    • description
      • Summary passage information of the blog post content
      • The part that matches the search term in the passage information is enclosed in <b> tags.
    • dateTime
      • Date the blog post was created
  • BlogBlock
    This class includes the business logic for sending API requests.
    • Currently, the response value is returned in JSON format. If you want to receive the response value in XML format, modify headers.setContentType(MediaType.***APPLICATION_JSON***) in the createHeaders() method to headers.setContentType(MediaType.***APPLICATION_XML***).

          private void processResponse(ResponseEntity<String> response) throws JsonProcessingException {
        ObjectMapper mapper = new ObjectMapper();

    • description
      • Description of the business or organization
    • telephone
      • This element does not return a value
      • It is included for backward compatibility.
    • roadAddress
      • Road address of the business or organization
  • LocalBlock
    This class includes the business logic for sending API requests.

1-2. Description

  • TTSBlock
    Class type that receives English text and stores it as an .mp3 voice file.
    • Constructor
      • Enter the path where the voice file will be stored (no instance can be created without this method)
    • synthesizeText
      • Receives text in String format and converts it to a voice file saved in the specified path
      • Enter the English text to be converted to voice in the text parameter with String type
      • Enter the desired country and gender as a TTSCountry type in the country parameter
  • TTSCountry
    Enum type that designates the country and gender to record the voice.
    • US__ALPHABET_MALE
      • Saves the voice file with the American male voice specified by the TTS API
    • US__ALPHABET_FEMALE
      • Saves the voice file with the American female voice specified by the TTS API
    • UK__ALPHABET_MALE
      • Saves the voice file with the British male voice specified by the TTS API
    • UK__ALPHABET_FEMALE
      • Saves the voice file with the British female voice specified by the TTS API
    • AU__ALPHABET_MALE
      • Saves the voice file with the Australian male voice specified by the TTS API
    • AU__ALPHABET_FEMALE
      • Saves the voice file with the Australian female voice specified by the TTS API
    • KR__ALPHABET_MALE
      • Saves the voice file with the Korean male voice specified by the TTS API
    • KR__ALPHABET_FEMALE
      • Saves the voice file with the Korean female voice specified by the TTS API

1-3. Usage Example

  • Connecting to the TTS API and setting the voice file save path
    • Create a TTSBlock instance
      TTSBlock ttsBlock = new TTSBlock("your_path");
  • Entering a voice file
    • Enter the String type text and TTSCountry type value with the synthesizeText function
      ttsBlock.synthesizeText("Hello, welcome to South Korea", TTSCountry.UK_B_MALE);

Text API

1. Text API

1-1. Description

  • TextBlock
    Class type that receives text and performs various text inspections.
    • Constructor
      • Creates an instance by receiving text as a String type in the content parameter.
    • setContent
      • Sets the content value by limiting the length of the string.
    • setLengthLimit
      • Sets the maximum length allowed for the string.
    • getContentLength
      • Returns the length of the current content string.
    • isLengthOver
      • Checks if the content length is longer than the given length.
    • onlyKorean
      • Checks if the content consists only of Korean characters.
    • isOnlyLowercaseEnglish
      • Checks if the content consists only of lowercase English letters.
    • isOnlyUppercaseEnglish
      • Checks if the content consists only of uppercase English letters.
    • isOnlyEnglish
      • Checks if the content consists only of English letters.
    • isBlank
      • Checks if the content is empty.
    • containsSpecialCharacters
      • Checks if the content contains special characters.
    • doesNotContainSpecialCharacters
      • Checks if the content does not contain special characters.
    • containsNoCharacters
      • Checks if the content is empty.
    • matchesPattern
      • Checks if the content matches the given regular expression.

1-2. Usage Example

  • Connecting to the Text API and processing text
    • Create a TextBlock instance
    TextBlock textBlock = new TextBlock("Hello, DevWay!");
  • Setting a string
    • Set the content value with the setContent method
    textBlock.setContent("Welcome to DevWay");
  • Checking the string length
    • Get the string length with the getContentLength method
    int length = textBlock.getContentLength();
System.out.println("Content Length: " + length);
  • Checking the string pattern
    • Check if the string is Korean with the onlyKorean method
    boolean isKorean = textBlock.onlyKorean();
System.out.println("Is Korean: " + isKorean);
  • Checking if it contains special characters
    • Check if it contains special characters with the containsSpecialCharacters method
    boolean hasSpecialChars = textBlock.containsSpecialCharacters();
System.out.println("Contains Special Characters: " + hasSpecialChars);

Image API

1. Image API

1-1. Description

  • ImageBlock
    Class type that performs functions such as saving or converting image files locally.
    • Constructor
      • Creates an instance by receiving imagePath and imageUrl parameters.
    • downloadImage
      • Receives an image URL and saves it as an image file locally.
    • convertPathToUrl
      • Converts the local image file path to a URL.
    • imageToBytes
      • Converts an image file to a byte array.
    • saveBytesToImageFile
      • Saves the byte array as an image file.

1-2. Usage Example

  • Connecting to the Image API and processing images
    • Create an ImageBlock instance
    ImageBlock imageBlock = new ImageBlock();
  • Downloading an image
    • Save the image URL as a local file with the downloadImage method
    String imageUrl = "https://example.com/image.jpg";
String destinationPath = "local/path/to/image.jpg";
imageBlock.downloadImage(imageUrl, destinationPath);
  • Converting the local path to a URL
    • Convert the local path to a URL with the convertPathToUrl method
    String localPath = "local/path/to/image.jpg";
String url = imageBlock.convertPathToUrl(localPath);
System.out.println("Image URL: " + url);
  • Converting an image file to a byte array
    • Convert the image file to a byte array with the imageToBytes method
    String filePath = "local/path/to/image.jpg";
byte[] imageData = imageBlock.imageToBytes(filePath);
  • Saving a byte array as an image file
    • Save the byte array as an image file with the saveBytesToImageFile method
    String destinationPath = "local/path/to/saved_image.jpg";
imageBlock.saveBytesToImageFile(imageData, destinationPath);
